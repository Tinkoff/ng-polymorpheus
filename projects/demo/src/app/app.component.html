Takeaway

Polymorpheus allows you to write Design-agnostic code. In a sense, it is a lot like generics — you wrap your logic around what you receive from the user without dependence on what exactly you have received.

This concept frees you from flags and specific code for particular use cases. You only define boundaries of how your component looks like and implement actual work logic. Visual customization is delegated to whoever uses your component and is easily reused.

Other good examples that could benefit from this idea are popups, notifications, tooltips, dropdowns, error messages etc.
